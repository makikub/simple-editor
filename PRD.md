自分専用テキストエディタ 要件定義 MVP版

1. コンセプト

macOS 専用の軽量テキストエディタを作る。

主な目的は、さくらエディタの代替として、CSV、TSV、固定長ファイル、ログ、テキストファイルを安全に表示・編集できるようにすること。

特に重視するのは以下。

* 文字コードを壊さない
* 改行コードを壊さない
* CSVを表形式で見やすく表示・編集できる
* 固定長ファイルの桁・バイト位置を確認しやすい
* 保存時に文字欠落や文字化けを起こさない
* UIはできるだけシンプルにする
* macOSネイティブアプリとして自然に使えること

2. 対象OS

初期版は macOS のみ対応する。

Windows、Linux、Web版は対象外とする。

3. 実装技術方針

初期実装は Swift / SwiftUI による macOS ネイティブアプリとする。

必要に応じて AppKit を併用する。

採用理由

* macOS専用アプリとして自然に作れる
* ファイルオープン、保存、ドラッグ&ドロップ、メニュー操作との相性がよい
* Electron / Tauri より配布物を軽くしやすい
* 自分専用ツールとして保守しやすい
* ネイティブUIでシンプルな操作感を作りやすい
* AppKitを併用すれば高度なテキスト編集にも対応できる

技術構成

* Swift
* SwiftUI
* AppKit
* NSTextView
* SwiftUI Table または独自グリッド
* Swift標準ファイルAPI
* 必要に応じて Core Foundation の文字コードAPI

4. 想定利用シーン

主な利用シーンは以下。

* CSVファイルの内容確認
* CSVファイルの軽微な編集
* TSVファイルの内容確認
* 固定長ファイルの桁位置・バイト位置確認
* 固定長ファイルの軽微な編集
* Shift_JIS / CP932 ファイルの安全な編集
* UTF-8 / UTF-8 BOM ファイルの安全な編集
* ログファイルの確認
* SQL、JSON、XML、設定ファイルなどの確認
* さくらエディタ的な汎用テキスト編集

5. 対象ファイル

初期版で対象とするファイルは以下。

* .txt
* .csv
* .tsv
* .log
* .sql
* .json
* .xml
* 拡張子なしファイル
* 固定長ファイル

ファイル種別は拡張子から推定するが、ユーザーが表示モードを手動で切り替えられるようにする。

6. ファイルサイズ

初期版の想定ファイルサイズは、数MBから50MB程度までとする。

100MB以上の巨大ファイルを快適に編集することはMVPの対象外とする。

ただし、将来的に部分読み込みや仮想スクロールを導入できるよう、設計上は拡張余地を残す。

7. 表示モード

エディタは以下の3モードを持つ。

* テキストモード
* CSVモード
* 固定長モード

基本はテキストモードとする。

CSVモード、固定長モードは、同じファイルを別の見方で表示する補助ビューとして扱う。

画面上部に以下のような切り替えUIを設ける。

[テキスト] [CSV] [固定長]

CSVファイルを開いた場合でも、ユーザーはいつでもテキストモードで生の内容を確認できる。

8. 通常テキストモード

テキストモードでは、通常のテキストエディタとして以下を提供する。

必須機能

* ファイルを開く
* ファイルをドラッグ&ドロップで開く
* 保存
* 名前を付けて保存
* コピー
* 貼り付け
* Undo
* Redo
* 行番号表示
* 折り返し表示 ON / OFF
* 通常検索
* 正規表現検索
* 通常置換
* 文字コード表示
* 改行コード表示
* 未保存状態表示

初期版では対象外

* 複数ファイルタブ
* 正規表現置換
* 矩形選択
* 複数カーソル
* Git連携
* 差分表示
* 高度なシンタックスハイライト
* プラグイン機構

複数ファイルは初期版では扱わず、1ウィンドウ1ファイルとする。

9. テキストエディタ実装方針

テキストエディタ部分は、SwiftUI の TextEditor ではなく、AppKit の NSTextView を SwiftUI にラップして実装する。

理由は、TextEditor はシンプルだが、高度なテキストエディタ用途では制御しにくいため。

必要な制御は以下。

* 行番号表示
* Undo / Redo
* 検索
* 正規表現検索
* 改行コード維持
* カーソル位置取得
* 選択範囲取得
* 固定長ルーラーとの連動
* ステータスバーとの連動

10. 文字コード対応

初期版で対応する文字コードは以下。

* UTF-8
* UTF-8 with BOM
* CP932 / Shift_JIS

EUC-JP、UTF-16、ISO-2022-JP は初期版では対象外とする。

ただし、将来的に追加できるよう、文字コード判定・変換処理は独立したモジュールとして設計する。

11. 文字コード処理方針

文字コード処理は独立モジュールとして実装する。

候補モジュールは以下。

FileEncodingDetector
FileDecoder
FileEncoder

Swift標準の String.Encoding.shiftJIS だけで、Windows由来のCP932を完全に安全に扱えるかは検証が必要。

特に以下の文字で差分が出る可能性がある。

* 丸付き数字
* ローマ数字
* 機種依存文字
* Windows拡張文字

必要に応じて、Core Foundation の encoding API などを利用する。

12. 文字コード自動判定

ファイル読み込み時に文字コードを自動判定する。

判定対象は以下。

* UTF-8
* UTF-8 with BOM
* CP932 / Shift_JIS

読み込み後、ステータスバーに判定結果を表示する。

例。

Encoding: CP932

自動判定に失敗した場合、またはデコード不能なバイト列が存在する場合は警告を表示する。

デコード不能な文字を勝手に � に置換して保存してはならない。

13. デコード不能時の扱い

デコード不能なバイト列がある場合、以下の動作とする。

* ファイルを警告付きで開く
* 問題箇所を可能な範囲で示す
* 保存はブロックする
* ユーザーが文字コードを変更して再読み込みできるようにする

保存ブロック時には理由を明示する。

例。

このファイルにはCP932としてデコードできないバイト列があります。
文字欠落を防ぐため保存できません。

14. 改行コード対応

対応する改行コードは以下。

* LF
* CRLF
* CR

読み込み時に主な改行コードを自動判定する。

保存時は、原則として読み込み時の改行コードを維持する。

混在改行がある場合は、ステータスバーまたは警告領域に表示する。

例。

改行コード: CRLF

混在時。

改行コード: CRLF中心 / LF混在あり

15. 保存仕様

保存時は安全性を最優先する。

基本仕様

* 元の文字コードを維持する
* 元の改行コードを維持する
* 対象文字コードで表現できない文字がある場合は保存をブロックする
* デコード不能文字がある場合は保存をブロックする
* 保存時は atomic save を行う
* 保存前バックアップは設定で ON / OFF 可能にする
* 初期設定ではバックアップ ON とする

atomic save

直接上書きせず、一時ファイルに書き込んでから置換する。

保存失敗時に元ファイルを壊さないことを目的とする。

流れは以下。

1. 保存対象文字列を生成
2. 対象文字コードでエンコード可能か検証
3. .bak を作成
4. 一時ファイルへ書き込み
5. 書き込み成功後に元ファイルと置換
6. 失敗時は元ファイルを残す

バックアップ

保存時に同じディレクトリへ .bak ファイルを作成できるようにする。

例。

sample.csv
sample.csv.bak

初期設定ではバックアップ作成を ON とする。

16. CSVモード

CSVモードでは、CSV / TSV を表形式で表示・編集できる。

対象

* CSV
* TSV
* 拡張子なしの区切り文字ファイル

区切り文字

区切り文字は自動判定する。

判定対象は以下。

* カンマ
* タブ

拡張子による初期推定は以下。

* .csv はカンマ優先
* .tsv はタブ優先
* 拡張子なしは内容から推定

ユーザーは区切り文字を手動で切り替えられる。

17. CSVパース仕様

Excel由来のCSVを基本的に壊さず扱えることを目標とする。

以下に対応する。

* ダブルクォートで囲まれたセル
* カンマを含むセル
* セル内改行
* ダブルクォートのエスケープ
* 空セル
* 末尾カンマ
* ヘッダー数とデータ列数が異なる行

例。

id,name,memo
1,"山田,太郎","カンマを含む"
2,"佐藤","1行目
2行目"
3,"鈴木","彼は""OK""と言った"

上記を正しく3列として扱う。

単純な split(",") による実装は禁止する。

CSVパースは独自実装またはSwift製ライブラリを検討する。

18. CSVヘッダー

CSVモードでは、ヘッダーあり / ヘッダーなしの両方に対応する。

ヘッダーあり

1行目をヘッダーとして扱う。

ヘッダー行は固定表示する。

データ領域だけ縦スクロール可能にする。

ヘッダーなし

列名は Excel 風に表示する。

例。

A, B, C, D, E

ユーザーはヘッダーあり / なしを切り替えられる。

19. CSV表示仕様

CSVモードでは以下を提供する。

* 表形式表示
* ヘッダー固定
* 行番号表示
* データ領域のみスクロール
* 列幅自動調整
* 列幅手動変更
* セル選択
* セル編集
* CSV全セル検索
* CSV列指定検索
* 検索結果一覧
* 表示上のフィルタ

初期版では、実データを並び替えて保存するソート機能は対象外とする。

フィルタは表示上のみとし、保存時に行順を変更しない。

20. CSVグリッド実装方針

CSVグリッド部分は SwiftUI の Table または独自グリッドで実装する。

初期候補は以下。

* SwiftUI Table
* LazyVGrid
* ScrollView + 独自セル表示
* 必要に応じて NSScrollView + AppKit カスタムビュー

MVPでは、巨大なExcel相当グリッドではなく、数MB〜50MB程度のCSVを扱える範囲を狙う。

21. CSVセル編集

セル編集の操作感は Excel に近づける。

初期版では以下を対応する。

* ダブルクリックで編集開始
* Enterで編集開始
* Escで編集キャンセル
* Enterで編集確定
* Tabで次セルへ移動
* Shift + Tabで前セルへ移動
* 矢印キーでセル移動

直接入力でセル上書きする操作は、誤操作の可能性があるため後回しとする。

22. CSVコピー・貼り付け

初期版で以下に対応する。

* 単一セルコピー
* 複数セルコピー
* Excelへ貼り付け可能な形式でコピー
* Excelから複数セル貼り付け
* 行コピー

列コピーはできれば対応するが、MVP必須からは外す。

23. CSV行・列操作

初期版で対応する。

* 行追加
* 行削除

初期版では以下を対象外とする。

* 列追加
* 列削除
* 列並び替え
* 実データのソート保存

列操作は全行に影響し、ファイル破壊リスクが上がるため後回しとする。

24. CSV保存仕様

CSVモードで編集した場合、保存時は以下とする。

* 未編集行は元のテキストをそのまま維持する
* 編集した行だけCSVとして再生成する
* 保存時は元の文字コードを維持する
* 保存時は元の改行コードを維持する
* 対象文字コードで表現できない文字があれば保存をブロックする

CSV全体を無条件に再生成してはならない。

これにより、未編集行のクォート有無、空白、末尾カンマなどを可能な範囲で維持する。

25. 固定長モード

固定長モードは、初期版では表形式編集までは行わない。

目的は、固定長ファイルを安全に確認・軽微編集するための補助表示とする。

初期版で提供する機能

* 文字位置ルーラー
* バイト位置ルーラー
* 固定間隔ガイド
* 任意位置ガイド
* カーソル位置の文字位置表示
* カーソル位置のバイト位置表示
* 行ごとの文字数表示
* 行ごとのバイト長表示
* 行のバイト長変化の警告
* 保存前の固定長チェック

26. 固定長ルーラー

固定長モードでは、上部にルーラーを表示する。

例。

byte: 000000000111111111122222222223333333333
      123456789012345678901234567890123456789
001 | 0001山田太郎      20260630A01
002 | 0002佐藤花子      20260630A02

初期版では、文字位置とバイト位置の両方を表示する。

CP932バイト位置表示は、必要になったときに拡張できる設計にする。

27. 固定長ガイド

固定長モードでは、以下のガイド表示に対応する。

* 10文字ごとの固定ガイド
* 10バイトごとの固定ガイド
* 任意位置の手入力ガイド

任意位置はカンマ区切りで入力できる。

例。

8,20,32,40,64

初期版では、JSON / YAML のカラム定義ファイル読み込みは対象外とする。

28. 固定長編集時の警告

固定長モードでは以下を警告する。

* 行の文字数が変わった
* 行のバイト長が変わった
* 指定ガイド幅を超えた
* 保存前チェックで行長が揃っていない

固定長ファイルは編集で崩れやすいため、警告を明確に表示する。

ただし、初期版では警告にとどめる。保存ブロックするかどうかは設定で切り替えられるようにする。

29. 検索・置換

初期版では以下に対応する。

テキストモード

* 通常検索
* 正規表現検索
* 通常置換
* 検索結果一覧

CSVモード

* 全セル検索
* 列指定検索
* 検索結果一覧

固定長モード

* 通常検索
* 正規表現検索

初期版では以下を対象外とする。

* 正規表現置換
* 複数ファイル横断検索
* grep機能
* 固定長範囲指定検索

30. ステータスバー

画面下部にステータスバーを表示する。

表示項目は以下。

* 文字コード
* 改行コード
* ファイルサイズ
* 行数
* 現在行
* 現在桁
* 選択文字数
* 現在位置のバイト位置
* CSV列数
* CSV行数
* 未保存状態
* 読み取り専用状態
* デコード警告
* 保存ブロック理由

表示例。

CP932 | CRLF | 2.4MB | 行 120/3000 | 桁 35 | byte 42 | 未保存

CSVモードの場合。

CP932 | CRLF | CSV | 12列 | 3000行 | 行 120 | 列 name | 未保存

31. ドラッグ&ドロップ

ファイルをアプリへドラッグ&ドロップして開けるようにする。

初期版では1ファイルのみ対応する。

複数ファイルがドロップされた場合は、先頭の1ファイルを開くか、警告を出す。

既に未保存のファイルが開かれている場合は、保存確認を表示する。

32. 自動保存・クラッシュ復元

業務ファイルを勝手に上書きしないため、自動保存は行わない。

ただし、クラッシュ復元用の一時データは保持する。

初期版仕様

* 元ファイルへの自動保存はしない
* 編集中の内容を一時領域に保持する
* アプリ異常終了後、次回起動時に復元候補を表示する

36. 主要モデル案

OpenedFile

開いているファイルの状態を表す。

保持項目。

* ファイルURL
* ファイル名
* 元バイト列
* 表示用テキスト
* 文字コード
* 改行コード
* ファイルサイズ
* 未保存状態
* デコード警告
* 読み取り専用状態

EncodingInfo

文字コード情報を表す。

保持項目。

* 判定された文字コード
* BOM有無
* 判定信頼度

LineEndingInfo

改行コード情報を表す。

保持項目。

* 主改行コード

CSVDocument

CSV表示用の構造を表す。

保持項目。

* 区切り文字
* ヘッダーあり / なし
* ヘッダー配列
* 行配列
* 元行テキスト
* 編集済み行ID
* パース警告

CSVRow

CSVの1行を表す。

保持項目。

* 行番号
* セル配列
* 元テキスト
* 編集済みかどうか
* パース上の開始位置
* パース上の終了位置

FixedWidthGuide

固定長ガイド設定を表す。

保持項目。

* 文字位置ガイド
* バイト位置ガイド
* 任意位置ガイド
* ガイド単位
* 保存前チェック設定

40. 最重要品質要件

このエディタで最も重要なのは、便利機能よりもファイルを壊さないこと。

そのため、以下を品質要件として明示する。

* 文字コードを勝手に変換しない
* 改行コードを勝手に変換しない
* デコード不能文字を勝手に置換しない
* 保存時に表現不能文字があれば保存しない
* CSV未編集行はできるだけ元のテキストを維持する
* 保存失敗時に元ファイルを壊さない
* 固定長ファイルでは行長変化を警告する
* 自動保存で元ファイルを勝手に上書きしない
